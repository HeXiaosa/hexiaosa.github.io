Java 内存管理部分

字符串内存分析

String s1 = "a";
String s2 = "b";
String s3 = "a" + "b";
System.out.println("ab == s1+s2" + ("ab" == s1+s2));
System.out.println("ab == s3" + ("ab" == s3));
输出结果：
false
true
直接用引号包裹的字符串相加产生的新对象会放到常量池中，对象相加的不会放到常量池中。


public static final String A = "ab"; // 常量A
public static final String B = "cd"; // 常量B
A+B 会放到常量池中，因为 A, B 都是常量，在类编译时已经确定了。


public static final A;
public static final B;
static {
    A = "a";
    B = "b";
}
A+B 不会放到常量池中，虽然被 final 修饰，但是在后来被赋值，并不是在一开始编译时就确定的。

intern()方法表示，如果字符串在常量池中存在，就取常量池中的。

String hello = "hello", lo = "lo";
hello == "hel" + "lo"; //true
hello == "hel" + lo; //false
hello == ("hel" + lo).intern();  //true

Integer 内存分析

Integer i = new Integer(23);
使用 Integer 用以上方式创建对象时，当数值在 -128~127 之间，对象都是在存在于常量池中的，两个相同的数值 new Integer() "==" 比较结果是 true.
其它范围的数值不在常量池中，每次 new Integer() 都是一个新的对象，"==" 比较结果是 false.
