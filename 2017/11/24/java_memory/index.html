Java 内存管理部分

Veröffentlicht am 2017-11-24

1. JVM 内存

JVM 会用一段空间来存储程序运行时的数据和相关信息，这段空间一般称作 Runtime Data Area. 包含 Java 堆，方法区，（和线程私有的）程序计数器，虚拟机栈，本地方法栈。
Java 堆：用于存放 new 出的对象，垃圾回收也主要就是针对这一块；
方法区主要存放 静态变量，常量，方法区就包含了常量池；
本地方法栈存放的就是 native 方法；
虚拟机栈中有一个局部变量表，存放的是基本类型和对象的引用；虚拟机栈用于存方法在执行的内存区，每个方法执行都会创建栈针，方法执行的过程，也就是虚拟机栈的入栈出栈。

2. 字符串内存分析

输出结果：

直接用引号包裹的字符串相加产生的新对象会放到常量池中，对象相加的不会放到常量池中。

intern()方法表示，如果字符串在常量池中存在，就取常量池中的。

3. Integer 内存分析

定义一个 Integer 类型，赋值方式

  1. Integer i = new Integer(10);
  2. Integer i = 10;

对于 -128~127 之间的数，使用第二种赋值方式时如果值相同，则两个变量 “==” true;
其它范围的数，两种赋值方式比较结果都为 false.

原因：
-128~127 之间的数会存储在常量池中，比较的直接就是存在在常量池中的对象，而其它数都是新建的对象。
