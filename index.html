HTTPS 加密传输

Veröffentlicht am 2019-04-09

HTTPS 相对于 HTTP 是安全的，它是加密传输的，那么它是怎么加密传输呢，数字证书又是什么，有数字证书就一定是可信任的吗，下面来一一解答

什么是数字证书

数字证书可以是组织颁发，也可以是自己颁发的，组织颁发的证书，客户端会信任该服务，直接访问，自己颁发的证书，会弹框，用户确认后才可以访问。

HTTPS 加密中使用了非对称加密，数字证书中就包含了非对称加密所需的公钥和私钥。

tip :
CA : 颁发证书的机构
CA 证书：CA 颁发的证书，就是我们说的数字证书，包括持有者身份，CA机构的签名，公钥和私钥。CA机构的签名用于验证证书的有效性。

HTTPS 是怎么加密的

对称加密是通过一个秘钥来加密和解密的，秘钥在客户端和服务端之间传输，如果被劫持，还是会被破解；
非对称加密是通过公钥加密，私钥解密的，服务端把公钥传给客户端，然后客户端用公钥加密传输的内容，传给服务端，服务端用私钥解密内容，这样就算被劫持，因为没有私钥，也无法解密。

所以使用对称加密是不安全的，非对称加密是安全的，但是非对称加密效率比较低，HTTPS 就采用了对称加密和非对称加密结合的方式。

具体过程用这篇文章里找到的这张图描述更清晰 （https://juejin.im/post/5a4f4884518825732b19a3ce）

这个过程用文字描述：
服务端把公钥给客户端，客户端在验证有效后，生成一个随机值，这个随机值就是对称加密的秘钥，公钥加密这个秘钥传给服务端，服务端用私钥解密，得到对称加密的秘钥，然后用这个秘钥加密要传输的内容。

HTTPS 证书相关

上面基本搞清楚加密传输的流程，但是加密过程中使用的数字证书，客户端是怎么验证这个证书是可信任的呢。
客户端通过 TSL 协议（SSL发展过来的，3.0改为TSL）来验证，TSL 的作用包含：

加密
证书验证
验证消息完整性

加密过程在上面已经介绍过了，这里只说下证书验证，证书验证就是从数字证书中取出颁发者，源头的颁发者非常少，于是就是看颁发者是否可信，颁发者可信，这个证书也就是可信的。

View 坐标体系和 View 的滑动

Veröffentlicht am 2019-01-07

View 坐标介绍

View 坐标表示有以下几种， getLeft, getTranstationX, getScrollX, getRawX, getX. 那么这几种表示有什么区别呢

  1. getLeft 表示 View 的实际坐标，View 的实际左上角距离父 View 左上角的水平距离。
  2. getTranslationX 表示 View 与父 View 的水平偏移量，可以通过设置 view.setTranslationX(*) 来移动 view, 改变的就是 translationX.
  3. getScrollX 获取的是 view 的内容相对 view 移动的水平距离。
  4. getRawX 表示 View 的实际左上角距离屏幕左上角的水平距离。
  5. getX = getLeft() + getTranslationX

注意：onTouchEvent 随手指移动时不应使用 view.getX() 来计算移动距离，应该使用 view.getRawX() 计算移动距离，因为 view.getX() 获取的是 view 左上角与 父 view 左上角的距离，但是 view 移动，view.getX() 也会随着变化，所以如果使用 view.getX() 随手指移动 view 会抖动。

事件分发概述

Android 中触摸事件主要由 dispatchTouchEvent, onInterceptTouchEvent, onTouchEvent 来控制。

触摸屏幕时，首先会触发 ViewGroup 的 dispathTouchEvent 方法，自定义 View 时，如果它在 ACTION_DOWN 返回 false, 则表示不再继续向下执行，事件到此结束，如果返回了 true, 则下次继续执行 dispatchTouchEvent 方法，返回 super.dispatchTouchEvent(ev) 则会执行 onInterceptTouchEvent/onTouchEvent.

View 是没有 onInterceptTouchEvent 方法的，因为事件传递到 View 之后是一定交给自己处理了。ViewGroup 的 onInterceptTouchEvent 方法 返回 true 表示 要拦截事件，给自己处理，返回 false 表示不拦截，传递给子 View 处理，执行子 View 的 dispatchTouchEvent 方法。

执行到 onTouchEvent 方法，返回 true 表示不再给它的 ViewGroup 处理，自己处理完就结束，返回 false 表示自己可以做一些操作，再执行它的 ViewGroup 的 onTouchEvent 方法。
