Java 集合简括

Veröffentlicht am 2018-01-24

  1. Map
    1.1 HashMap
    以键值对的形式保存，数据结构为数组加链表，不同的 key 所生成的 hash 可能对应的 index 是相同的。
    hash 地址对应的数组 index 相同的话，就用头插法将该元素放到 index 位置的链表上。
    同一个 key 只有一个 value, 如果对于这个 key 再 put 一个 value 的话，就覆盖掉原来的 value, 并 put 方法返回原 value.
    其实际存的用 HashMapEntry 对象数组。在 Java 源码中是 Node, 它们的属性和方法是一样的。
    HashMap 的数组长度取 2 的 n 次方的原因：其二进制数是 1 后面都是零，减 1 后全是 1, 由 hash 转换 index 时拿 length-1 和 hash地址做位于操作，这样得到的就是余数。
    1.2 HashTable
    与 HashMap 相比，读写数据的方法都加了一个 synchronized.

  2. List
    2.1 ArrayList
    以数组形式保存，初始化时可以指定数组长度，不指定的话有默认的长度，每次 add 时检测数组长度是否够用，不够用的话即扩容。
    查找效率高，插入，删除都需要移动该元素后面的所有元素。
    2.2 LinkedList
    以双链表形式保存，在头尾插入删除效率高；查找指定位置的会根据要查找的位置来判断从头还是尾开始查。

  3. Set
    3.1 HashSet
    无顺序，无重复元素。如果插入的元素已存在，会覆盖已存在的那个。使用的是 HashMap 的 key 来存放。
    3.2 TreeSet
    有顺序，无重复元素。如果 add 的元素已存在，会覆盖已存在的那个。实际存的就是 TreeMap 的 key.
